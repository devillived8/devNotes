Введение

HTML (HyperText Markup Language) — это код, который строит веб-страницы. Каждый сайт, который ты открываешь в браузере, внутри состоит из HTML-тегов. Они говорят браузеру: «Этот текст будет заголовком, этот — ссылкой, а это — картинка». Сам по себе HTML не умеет ни считать, ни думать, ни реагировать на клики. Его задача — структура. Всё остальное делают CSS (внешний вид) и JavaScript (логика).

История: как всё начиналось

В 1989 году учёный Тим Бернерс-Ли работал в европейской лаборатории CERN. Учёным нужно было обмениваться документами, но каждый раз пересылать файлы и следить за версиями было мучительно. Тим придумал систему, в которой документы связаны между собой ссылками, — так родилась идея Всемирной паутины. Для этой системы нужен был язык разметки. Тим взял за основу SGML (сложный стандарт для документов) и упростил его до минимума. Так в 1991 году появился первый HTML — с парой десятков тегов, без картинок и таблиц. Первый в мире сайт (до сих пор работает) — http://info.cern.ch. Дальше HTML развивался скачками: HTML 2.0 (1995) — появились формы HTML 3.2 (1997) — таблицы и скрипты HTML 4.01 (1999) — разделение структуры и оформления (стили вынесли в CSS) XHTML (2000-е) — попытка сделать HTML строже (не взлетела) HTML5 (2014) — современный стандарт: семантика, видео, аудио, canvas

Что изменил HTML5

До HTML5 верстали примерно так:

<div class="header">Шапка</div>
<div class="nav">Меню</div>
<div class="content">Контент</div>

Браузеру было всё равно — везде просто div. HTML5 ввёл семантические теги, которые не меняют внешний вид, но объясняют, что где находится:

  • <header> — шапка сайта или раздела
  • <nav> — навигация
  • <main> — главный контент (уникальный для страницы)
  • <article> — независимая статья или пост
  • <section> — смысловой раздел
  • <aside> — боковая панель (дополнительный контент)
  • <footer> — подвал
  • Семантика - разработчикам — код читается как книга
  • SEO — поисковики лучше понимают структуру
  • Скринридерам — незрячие люди быстрее ориентируются

Как устроен HTML-документ

Любой HTML-документ имеет обязательную структуру — скелет, с которого всё начинается: 

<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <title>Заголовок страницы</title>
</head>
<body>
  <h1>Привет, мир!</h1>
  <p>Это мой первый сайт.</p>
</body>
</html>

Разберём по косточкам:

  • <!DOCTYPE html> — объявление типа документа. Говорит браузеру: "используй HTML5".
  • <html> — корневой элемент, обёртка для всей страницы. Атрибут lang указывает язык.
  • <head> — контейнер для служебной информации: кодировка, заголовок вкладки, стили, мета-теги.
  • <meta charset="UTF-8"> — задаёт кодировку текста (чтобы русский отображался нормально).
  • <title> — заголовок страницы, который видно во вкладке браузера.
  • <body> — всё, что видит пользователь: текст, картинки, кнопки.

Всё, что ты видишь на сайте — внутри <body>. Остальное — техническая часть.

Формы

Формы нужны, чтобы пользователь мог отправить данные на сервер: зарегистрироваться, написать сообщение, оставить отзыв.

Базовая структура формы:

<form action="/submit" method="post">
  <!-- Поля формы будут здесь -->
</form>
  • action — адрес, куда улетают данные
  • method — способ отправки (обычно post или get)

Основные элементы форм

  • <input> — универсальное поле. Тип задаётся атрибутом type:
    • type="text" — обычный текст
    • type="email" — проверяет, что введён email
    • type="password" — скрывает символы
    • type="checkbox" — галочка
    • type="radio" — переключатель
    • type="submit" — кнопка отправки
  • <textarea> — многострочное поле для текста
  • <select> + <option> — выпадающий список
  • <button> — кнопка
  • <label> — подпись к полю (кликабельная)

Важно для доступности

Каждое поле должно иметь <label>, связанный через for="id": 

<label for="username">Имя:</label>
<input type="text" id="username" name="username">

Тогда:

  • Клик по подписи активирует поле
  • Скринридер прочитает название поля

Пример формы 

<form>
  <label for="name">Имя:</label>
  <input type="text" id="name" name="name">

  <label for="email">Email:</label>
  <input type="email" id="email" name="email">

  <label for="message">Сообщение:</label>
  <textarea id="message" name="message"></textarea>

  <button type="submit">Отправить</button>
</form>
